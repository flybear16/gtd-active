# GTD 技能

基于 David Allen 的《Getting Things Done》方法论，自动化管理任务和想法。

## 核心理念

> 你的大脑是用来思考的，不是用来存储的。

GTD 将一切事务从大脑中清空，放入可信赖的系统，让你专注于执行。

---

## 五步流程

| 步骤 | 动作 | 说明 |
|------|------|------|
| 1. 收集 | 📥 Capture | 将所有想法、任务、信息放入收件箱 |
| 2. 理清 | 🔍 Clarify | 判断每个项目的性质和下一步 |
| 3. 组织 | 📂 Organize | 分类到合适的清单 |
| 4. 回顾 | 🔄 Reflect | 定期检查各清单，保持系统更新 |
| 5. 执行 | ⚡ Engage | 根据情境选择合适的行动 |

---

## 清单体系

### 📥 收件箱
**原始输入，未处理**
- 所有进入系统的内容首先放这里
- 定期清空（建议每天）

### 🎯 项目
**需要多个步骤完成的目标**
- 如："上线新产品"、"完成年度报告"
- 每个项目应有明确的下一步行动

### ✅ 下一步行动
**可立即执行的具体任务**
- 如："写产品需求文档"、"打电话给客户"
- 按情境可标记：@办公室、@家、@电话、@电脑

### ⏰ 日程
**特定时间执行的任务**
- 如："周五14:00开会"、"下周一提交报告"
- 有明确的时间限制

### ⏳ 等待
**委托他人或等待外部条件**
- 如："等设计团队提供设计稿"、"等客户反馈"
- 需标注等待对象和时间

### 💭 将来/也许
**暂时不执行但想保留的想法**
- 如："学习一门乐器"、"去冰岛旅行"
- 定期回顾，可能转为项目

### 📎 参考资料
**非行动性信息，需要保存**
- 如："会议笔记"、"项目背景资料"
- 可搜索、可检索

### 🗄️ 存档
**已完成的任务和项目**
- 保留历史记录
- 用于回顾和复盘

## 自动分类规则

当用户输入任务时，根据关键词和语境自动判断分类：

### 判断流程

```
输入内容
    │
    ▼
是否是可行动的？──── 否 ───→ 📎 参考资料 / 💭 将来也许
    │
    是
    │
    ▼
是否需要多个步骤？──── 是 ───→ 🎯 项目 + 拆解下一步行动
    │
    否
    │
    ▼
是否有明确时间？──── 是 ───→ ⏰ 日程
    │
    否
    │
    ▼
是否需要他人完成？──── 是 ───→ ⏳ 等待
    │
    否
    │
    ▼
✅ 下一步行动
```

### 关键词识别

| 清单 | 触发关键词 |
|------|-----------|
| ⏰ 日程 | 周一/周二/...、明天、下周、X月X日、X点、之前、截止 |
| ⏳ 等待 | 等、等待、某人、让、委托、需要XX |
| 💭 将来/也许 | 想要、希望、如果、也许、以后、有机会、考虑 |
| 📎 参考资料 | 笔记、记录、资料、信息、文档、说明 |
| 🎯 项目 | 项目、计划、目标、完成、上线、推进 |

## 使用方式

### 命令行

```bash
# 添加任务（自动分类）
~/.copaw/scripts/gtd.sh add "周五下午3点开会"

# 添加到指定清单
~/.copaw/scripts/gtd.sh add "写需求文档" next_actions

# 查看清单
~/.copaw/scripts/gtd.sh list
~/.copaw/scripts/gtd.sh list inbox
~/.copaw/scripts/gtd.sh today

# 完成任务
~/.copaw/scripts/gtd.sh done <任务ID>

# 处理收件箱
~/.copaw/scripts/gtd.sh process

# 周回顾
~/.copaw/scripts/gtd.sh review
```

### 对话中使用

当用户说：
- "我有一个想法..." / "记住..." / "别忘了..." → 添加到收件箱
- "今天要做什么" → 显示今日日程和下一步行动
- "我在等什么" → 显示等待清单
- "最近有什么项目" → 显示项目清单

Agent 会：
1. 自动识别任务类型
2. 应用 GTD 分类规则
3. 询问必要信息（时间、等待对象等）
4. 更新相应清单

## 在 Agent 中处理用户输入

当用户输入任何内容时，按以下逻辑处理：

### 1. 判断意图

```python
def classify_input(content):
    content_lower = content.lower()
    
    # 日程关键词
    if any(kw in content_lower for kw in ['周一', '周二', '明天', '下周', '点', '之前', '截止', '开会', '会议']):
        return 'calendar'
    
    # 等待关键词
    if any(kw in content_lower for kw in ['等', '等待', '需要', '委托']):
        return 'waiting'
    
    # 将来/也许关键词
    if any(kw in content_lower for kw in ['想要', '希望', '也许', '以后', '有机会', '考虑']):
        return 'someday'
    
    # 参考资料关键词
    if any(kw in content_lower for kw in ['笔记', '记录', '资料', '文档', '备忘']):
        return 'reference'
    
    # 项目关键词
    if any(kw in content_lower for kw in ['项目', '计划', '目标', '完成', '上线']):
        return 'projects'
    
    # 默认：下一步行动
    return 'next_actions'
```

### 2. 执行添加

```bash
~/.copaw/scripts/gtd.sh add "$content" "$list"
```

### 3. 反馈用户

```
✓ 已添加到 📥 收件箱
✓ 已添加到 ⏰ 日程
✓ 已添加到 ⏳ 等待
```

### 4. 检查项目关联（重要）

添加任务后，**务必检查是否能关联到现有项目**：

1. 先查看项目列表：`~/.copaw/scripts/gtd.sh list projects`
2. 对比任务内容与项目名称，看是否有关联
3. 如果可能关联，**主动询问用户确认**：
   ```
   这个任务看起来和项目「XXX」相关，要关联吗？
   ```
4. 用户确认后，编辑任务 JSON 文件，添加 `project_id` 字段：
   ```json
   {
     "id": "任务ID",
     "content": "任务内容",
     "project_id": "项目ID",
     ...
   }
   ```

**关联规则：**
- 任务内容包含项目名称或关键词 → 提示关联
- 用户明确指定项目 → 直接关联
- 不确定是否关联 → 询问用户
